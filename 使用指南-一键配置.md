# chen-AI 测试包使用指南

## 📦 第一步：解压测试包

将 `chen-AI-测试包.zip` 解压到任意目录，例如：
- `D:\chen-AI-测试包`
- `C:\Users\你的用户名\Desktop\chen-AI-测试包`

## ⚙️ 第二步：一键配置（首次使用）

### Windows 用户

**方法一：双击运行（推荐）**

1. 进入解压后的目录
2. 双击 `setup.bat` 文件
3. 按照提示输入配置信息

**方法二：命令行运行**

1. 按 `Win + R`，输入 `cmd`，回车
2. 进入测试包目录：
   ```cmd
   cd D:\chen-AI-测试包
   ```
3. 运行配置脚本：
   ```cmd
   setup.bat
   ```

### 配置过程中需要输入的信息

1. **PostgreSQL postgres用户密码**
   - 这是你安装PostgreSQL时设置的postgres用户密码
   - 如果忘记了，可以重置（见下方常见问题）

2. **数据库名称**（可选，默认：`ai_qa_system`）
   - 直接回车使用默认值即可

3. **DeepSeek API Key**（可选）
   - 用于AI问答功能
   - 如果没有可以先留空，稍后在 `backend/.env` 文件中添加

### 配置完成后

配置脚本会自动：
- ✅ 生成 `backend/.env` 配置文件
- ✅ 创建数据库（如果不存在）
- ✅ 安装Python依赖
- ✅ 执行数据库迁移

## 🚀 第三步：一键启动

配置完成后，启动服务：

### Windows 用户

**方法一：双击运行（推荐）**

1. 双击 `start.bat` 文件
2. 会打开两个窗口：
   - 一个显示后端服务（Django）
   - 一个显示前端服务（Vite）

**方法二：命令行运行**

```cmd
start.bat
```

### 启动成功后

- **前端地址**: http://localhost:3000
- **后端API**: http://localhost:8000
- **管理后台**: http://localhost:8000/admin

## 🎯 第四步：开始使用

1. **打开浏览器**，访问 http://localhost:3000

2. **注册账号**
   - 点击"注册"按钮
   - 填写用户名、邮箱、密码
   - 注册成功后自动登录

3. **创建知识库**
   - 在首页输入知识库名称
   - 点击"创建"按钮

4. **上传BIM文件**
   - 选择知识库
   - 点击"上传 BIM"按钮
   - 选择 `.json` 或 `.ifc` 文件

5. **AI问答**
   - 选择知识库
   - 输入问题
   - 点击"提问"按钮

## 🛑 停止服务

关闭 `start.bat` 打开的两个窗口即可停止服务。

## ❓ 常见问题

### 1. PostgreSQL 密码忘记了怎么办？

**方法一：重置密码**

1. 打开命令提示符（管理员权限）
2. 进入PostgreSQL的bin目录（通常在 `C:\Program Files\PostgreSQL\14\bin`）
3. 执行：
   ```cmd
   psql -U postgres
   ```
4. 如果提示需要密码，可以修改PostgreSQL配置文件（需要管理员权限）

**方法二：使用Windows服务重置**

1. 按 `Win + R`，输入 `services.msc`
2. 找到 `postgresql-x64-14` 或类似的服务
3. 右键 → 属性 → 登录
4. 可以查看或修改服务账户

**方法三：重新安装PostgreSQL**

如果以上方法都不行，可以重新安装PostgreSQL并记住密码。

### 2. 提示"需要安装Python3"或"需要安装Node.js"

**安装Python：**
1. 访问 https://www.python.org/downloads/
2. 下载Python 3.9或更高版本
3. 安装时**勾选**"Add Python to PATH"
4. 安装完成后重启命令行

**安装Node.js：**
1. 访问 https://nodejs.org/
2. 下载LTS版本（推荐18.x或更高）
3. 安装完成后重启命令行

### 3. PostgreSQL服务未运行

**启动PostgreSQL服务：**

1. 按 `Win + R`，输入 `services.msc`
2. 找到 `postgresql-x64-14` 或类似的服务
3. 右键 → 启动

### 4. 端口被占用（8000或3000）

**解决方法：**

1. 关闭占用端口的程序
2. 或修改端口配置（需要修改配置文件）

### 5. 数据库连接失败

**检查项：**

1. PostgreSQL服务是否运行
2. `backend/.env` 文件中的 `DATABASE_URL` 是否正确
3. 数据库密码是否正确
4. 数据库是否已创建

### 6. 如何修改配置？

编辑 `backend/.env` 文件，修改后需要重启服务。

### 7. 如何查看日志？

- 后端日志：在运行 `start.bat` 的后端窗口中查看
- 前端日志：在运行 `start.bat` 的前端窗口中查看

## 📝 手动配置（如果一键配置失败）

如果一键配置脚本无法运行，可以手动配置：

### 1. 创建 `backend/.env` 文件

在 `backend` 目录下创建 `.env` 文件，内容：

```env
SECRET_KEY=test-secret-key-change-in-production
DATABASE_URL=postgresql://postgres:你的密码@localhost:5432/ai_qa_system
DEEPSEEK_API_KEY=你的DeepSeek-API-Key
DEBUG=True
ALLOWED_HOSTS=localhost,127.0.0.1,0.0.0.0
```

### 2. 创建数据库

打开命令提示符，执行：

```cmd
psql -U postgres
```

然后执行：

```sql
CREATE DATABASE ai_qa_system;
\c ai_qa_system
CREATE EXTENSION IF NOT EXISTS vector;
\q
```

### 3. 安装依赖

```cmd
cd backend
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements-minimal.txt
python manage.py migrate
```

### 4. 启动服务

**后端：**
```cmd
cd backend
.venv\Scripts\activate
python manage.py runserver 0.0.0.0:8000
```

**前端（新开一个窗口）：**
```cmd
cd frontend
npm install
npm run dev
```
