# 智知库首页通用能力聚焦蓝图

## 目标
- 突出“智知库 = 通用企业智能体底座”的核心价值，行业能力作为扩展包呈现。
- 优化首屏叙事与视觉层级，避免信息过载，建立清晰的阅读节奏。
- 在未登录状态下展示通用能力与价值证明，登录后扩展行业模块与数据工作台。
- 构建可持续的 JSON 导入能力：支持超大文件、异构结构（问答/知识条目/文档），并与向量化管道深度整合。

## 页面结构规划
1. **Hero / 价值主张**
   - 标题：`智知库 · 通用企业智能体中枢`
   - 副标题：`统一知识底座 + 智能编排 + 场景扩展`
   - 3 条 bullet（可带 icon）突出“安全可控 / 通用可扩 / 专属可复用”
   - CTA：`立即体验`（主按钮）+ `查看行业扩展`（次按钮）
   - 视觉：右侧保留弧线动效卡片，左侧文案区放大字号（48px）+ 适度留白

2. **平台能力概览（第二屏）**
   - 4 个 Feature Card，内容聚焦通用能力：
     1. `统一知识治理`
     2. `智能体编排`
     3. `安全与审计`
     4. `插件化扩展`
   - 每个卡片包含 icon、标题、两行描述以及“了解更多”轻按钮。

3. **核心机制 / 工作流（第三屏）**
   - 采用 3 步流程或线性时间轴：`数据接入 → 策略编排 → 多场景输出`
   - 列出关键要素（向量化策略、策略模板、权限域、监控）。
   - 加入一句“单一底座即可复用到不同部门/行业”。

4. **行业扩展专区（第四屏）**
   - 标题：`行业扩展包（基于通用底座）`
   - 卡片文案强调“在通用能力之上叠加行业要素”，而非独立产品线。
   - 默认展示金融、医疗；保留教育入口，可标记“正在发布”。

5. **数据工作台（登录用户）**
   - 仅在已登录状态显示，内容保持当前结构即可。
   - 未登录时隐藏该区块，以免稀释核心信息。

6. **知识导入板块**
   - 引导用户上传 PDF/文档以及批量 JSON。
   - 显示最近导入记录、进度状态（解析/向量化）。
   - 对 JSON 导入提供映射说明（支持 question/answer、content、metadata 等结构）。

## 设计原则
- **单一焦点**：Hero 只讲通用价值，不再混入行业/教育说明。
- **信息分段**：模块间最小留白 64px，背景交替（白/浅灰）形成节奏。
- **一致文案**：标题与副标题统一使用“智知库”“通用智能体”“扩展包”等关键词。
- **状态分离**：未登录 = 通用介绍，已登录 = 通用 + 行业扩展 + 工作台。
- **可扩展性**：所有卡片使用统一 slot 结构（icon / title / body / link）方便复用。
- **数据导入可靠性**：前端内建进度反馈，后端采用可重试、可观测的异步任务，任何条目失败都能追踪。

## JSON 导入自研蓝图

### 目标
- 支持导入 >100MB 的 JSON 数据集，包含问答、知识条目、通用文档等结构。
- 导入过程中不中断 HTTP 连接，用户可跟踪进度、查看失败条目。
- 自动触发知识库入库与向量化，并把原始字段映射到 metadata，便于溯源。

### 总体架构
1. **上传阶段**
   - 前端上传 JSON（或 JSONL、ZIP）至后端；超大文件可直接上传到对象存储。
   - API 接收后将文件保存到临时目录/对象存储，并创建 ImportJob 记录（状态 `pending`）。
   - 立即返回 Job ID，前端开始轮询。

2. **解析与入库阶段（异步）**
   - Celery Worker（或后台线程）读取文件，使用流式解析（ijson/JSONL）逐条生成文档。
   - `_normalize_json_entry` 负责字段映射：content/title/metadata，支持 question/answer/prompt/response 等。
   - 每写入 N 条就更新 ImportJob 的统计信息（total、created、skipped、progress%）。
   - 入库后将向量化任务排入队列，或在同一流程中延迟执行。

3. **向量化阶段（异步）**
   - 单独的 Worker 逐条向量化已入库文档，支持失败重试/跳过。
   - ImportJob 记录 vectorized_count、error_count，并为失败条目保留原因。

4. **前端交互**
   - 上传成功后进入“导入中”状态，显示进度条（总条数、已解析、已向量化）。
   - 轮询 `/import-jobs/<id>/` 查询状态；完成后可下载失败条目报告。
   - 支持取消导入或重试失败条目。

5. **运维与监控**
   - ImportJob 表记录全部元数据：文件路径、知识库 ID、提交人、耗时、状态、日志摘要。
   - Prometheus/ELK 监控队列长度、任务耗时、失败率；异常时报警。

### 关键模块
1. **REST API**
   - `POST /knowledge/import-jobs/`：创建导入任务，上传文件。
   - `GET /knowledge/import-jobs/<id>/`：查询状态与统计信息。
   - `DELETE /knowledge/import-jobs/<id>/`：取消未完成任务。

2. **数据模型**
   - `ImportJob`（id、knowledge_base、owner、status、total、created、vectorized、skipped、error_log、file_path、submitted_at、finished_at）。
   - `ImportJobEntry`（可选）：记录每条失败的 reason/原始数据指针。

3. **任务执行器**
   - `tasks.import_json(job_id)`：解析文件、创建 Document。
   - `tasks.vectorize_documents(job_id)`：对新增的 Document 批量向量化。

4. **字段映射策略**
   - `_normalize_json_entry` 扩展更多指令：标题截断、metadata 合并、字段白名单/黑名单。
   - 支持用户通过上传时附带 `field_mapping` 来覆盖默认行为（例如 `content = title + body`）。

5. **前端 UX**
   - 在“知识库”页面新增“导入记录”面板，展示最近的 import job。
   - “上传 JSON”按钮点击后显示分步进度（上传、解析、向量化），并在 504/网络失败时提供重试。
   - 对于超大文件，提示用户使用离线脚本上传到对象存储，再在页面中选择文件路径。

### 实施顺序
1. MVP：前端直接上传→后端同步执行（现状），适配 question/answer。
2. 阶段 1：上传后立即返回 Job ID，导入和向量化仍在一个任务里，但由后台线程执行；前端轮询进度。
3. 阶段 2：解析与向量化拆成两个任务，ImportJob 保存更多统计；引入失败条目报告。
4. 阶段 3：接入 Celery + 对象存储，支持分片/离线上传，完善监控与报警。

### 与首页联动
- 首页“知识导入”板块可展示最新的 Import Job 状态和入口按钮。
- 登录用户可在首页快速查看导入进度和失败条目，点击跳转到知识库详情。

这样自研蓝图能覆盖从上传、异步处理、可视化进度到运维监控的完整链路，既适配现有系统，又为未来扩展（多格式导入、自动字段映射、批量治理）打下基础。

执行该蓝图后，导师可一眼看到“智知库做的是通用智能体底座”，行业案例仅作为证明与扩展，页面叙事更加清晰集中。***
